## 📌 How to Contribute

### 1️⃣ Fork and Clone
1. Fork this repository by clicking the **Fork** button on the top right.
2. Clone your fork to your local machine:
   ```sh
   git clone https://github.com/your-username/mnzproject.git
   ```
3. Navigate to the project directory:
   ```sh
   cd mnzproject
   ```

### 2️⃣ Set Up the Environment
1. Install dependencies:
   ```sh
   npm install
   ```
2. Start the development server:
   ```sh
   npm run dev
   ```

### 3️⃣ Create a New Branch
Before making changes, create a new branch:
```sh
git checkout -b feature-your-feature-name
```

### 4️⃣ Make Your Changes
Modify the code, fix bugs, or improve the documentation.

### 5️⃣ Commit and Push
1. Stage your changes:
   ```sh
   git add .
   ```
2. Commit with a meaningful message:
   ```sh
   git commit -m "Add: Your concise commit message"
   ```
3. Push to your forked repository:
   ```sh
   git push origin feature-your-feature-name
   ```

### 6️⃣ Create a Pull Request
1. Go to the original repository: `https://github.com/username/mnzproject`
2. Click on **Pull Requests** > **New Pull Request**.
3. Select your branch and submit the PR with a clear description of your changes.

## 🎯 Contribution Guidelines
- Follow best practices for **React and JavaScript development**.
- Ensure code readability and proper formatting.
- Test your changes before submitting a PR.
- Write meaningful commit messages.
- Be respectful and supportive of the community.

## 📌 Commit Message Guidelines
Please use the following format when writing commit messages:

```
<type>: <short description>
```

### ✅ Allowed Commit Types
| Type     | Description |
|----------|-------------|
| `feat`   | Adding a new feature |
| `fix`    | Fixing a bug |
| `docs`   | Updating documentation |
| `style`  | Code style changes (formatting, indentation, etc.) |
| `refactor` | Code refactoring without changing functionality |
| `perf`   | Performance improvements |
| `test`   | Adding or updating tests |
| `chore`  | Maintenance tasks (dependency updates, build processes) |

### ✅ Examples
- `feat: Add like and save functionality for research papers`
- `fix: Resolve scrolling issue on mobile devices`
- `docs: Update README with installation instructions`
- `style: Improve button spacing for better UI`
- `refactor: Optimize recommendation algorithm`
- `test: Add unit tests for authentication system`
- `chore: Update dependencies and cleanup code`
